# 2026-06-23 Misread And Risk Register

## 목적

이 문서는 `C:\Users\abc20\Desktop\인터페이스 자동 생성기` 작업 중 발생했거나 다시 발생할 수 있는 오해, 위험 요소, 추가 확인 필요 사항을 기록한다.

## 해결된 오해와 수정 사항

### 1. 저장 형식 오해

초기에는 저장을 JSON 중심으로 해석할 위험이 있었다.

정정된 요구:

- 저장은 일반 사용자가 실행 가능한 사이트 폴더여야 한다.
- 최상위에는 `start-site.cmd`와 `output/`만 두는 구조가 맞다.
- `output/` 안에 React/Vite 앱과 정적 사이트 산출물을 포함한다.

현재 상태:

- `vite.config.js`의 `/api/save-project`가 이 구조로 저장한다.

남은 위험:

- 저장된 앱의 `npm install`은 사용자 PC의 npm/네트워크 상태에 영향을 받는다.

### 2. 네비게이션 메뉴 오해

초기에는 `Dashboard / Builder / Data / Settings` 같은 고정 메뉴를 네비게이션 기본값으로 두었다.

정정된 요구:

- 네비게이션 기본 메뉴는 없어야 한다.
- 캔버스에 추가된 다른 팔레트 기능마다 네비게이션 버튼이 하나씩 자동 생성되어야 한다.
- 편집 중에도 즉시 보여야 한다.

현재 상태:

- `src/lib/blockRuntime.js`가 네비게이션 외 블록을 기준으로 메뉴를 만든다.
- `src/components/BuilderCanvas.jsx`, `src/components/Preview.jsx`, 생성 React 앱, 정적 사이트가 같은 정책을 사용한다.

남은 위험:

- 블록 제목 또는 `navigationLabel`이 중복되면 네비게이션 버튼명이 중복될 수 있다.
- 중복 버튼을 누르면 같은 이름을 가진 블록 구분이 명확하지 않을 수 있다.
- 향후 `block.id` 기반 메뉴 타깃을 함께 저장하는 방식으로 개선하는 것이 안전하다.

### 3. 미리보기 방식 오해

오른쪽 패널 미리보기를 저장 전 확인용으로 충분하다고 해석할 위험이 있었다.

정정된 요구:

- `미리 보기` 버튼을 누르면 새 창에서 실제 캔버스 배치 결과를 보여줘야 한다.

현재 상태:

- `src/App.jsx`에서 새 창을 Blob URL로 연다.
- `src/lib/siteExport.js`의 `createPreviewDocument()`가 저장될 정적 사이트와 같은 형태의 HTML 문서를 만든다.

남은 위험:

- 브라우저 팝업 차단이 켜져 있으면 새 창이 열리지 않을 수 있다.
- 현재는 상태 메시지로 팝업 차단을 안내한다.

### 4. `about:blank` 빈 창 문제

새 창 미리보기에서 `about:blank` 빈 화면이 열렸다.

원인:

- `document.write()` 방식이 브라우저 조건에 따라 빈 창만 남길 수 있었다.
- `application/json` script 안의 프로젝트 JSON이 HTML escaped 상태가 되면 `JSON.parse`가 실패할 수 있었다.

현재 수정:

- Blob URL 방식으로 새 창을 연다.
- `serializeJsonForScript()`로 script-safe JSON을 넣는다.

남은 위험:

- 일부 브라우저 보안 정책에서 `blob:` 주소 팝업이 제한될 수 있다.
- 이 경우 별도 `/preview` 라우트 방식이 더 안정적일 수 있다.

### 5. 기능 블록 의미 전달 부족

초기 팔레트 이름/설명이 일반 사용자에게 기능 목적을 충분히 설명하지 못했다.

현재 수정:

- `src/data/blockCatalog.js`를 직관적인 한국어 명칭과 설명으로 정리했다.

남은 위험:

- 실제 사용자 테스트 없이 텍스트만 개선한 상태다.
- 비전문 사용자가 어떤 블록을 먼저 써야 하는지 안내하는 온보딩은 아직 없다.

### 6. 편집 캔버스와 저장 결과의 동작 차이

편집 캔버스, 오른쪽 미리보기, 새 창 미리보기, 저장된 React 앱, 정적 사이트가 각각 다른 구현 경로를 가진다.

현재 상태:

- 공통 정책은 `blockRuntime.js`와 `siteExport.js`에 최대한 맞췄다.

남은 위험:

- 같은 기능이 여러 렌더러에 중복 구현되어 있어 향후 수정 누락이 생길 수 있다.
- 특히 테이블/차트/탭 로직은 캔버스, 패널 미리보기, 생성 React 앱, 정적 스크립트에 각각 구현되어 있다.
- 장기적으로는 공통 렌더러 또는 공통 데이터/이벤트 스펙으로 통합해야 한다.

### 7. 정적 사이트의 HTML 삽입 위험

정적 사이트 `script.js`는 블록 제목/본문을 문자열로 조립해 `insertAdjacentHTML()`로 렌더링한다.

위험:

- 사용자가 블록 제목/본문에 HTML/스크립트성 문자열을 넣으면 생성된 정적 사이트에서 의도치 않은 HTML 삽입이 발생할 수 있다.

현재 완화:

- 프로젝트 JSON 자체는 script-safe 직렬화한다.

남은 위험:

- 렌더링 시 텍스트 escape가 완전하지 않다.
- 정적 렌더러는 DOM API의 `textContent` 기반 생성으로 바꾸는 것이 안전하다.

### 8. localStorage 기존 데이터 위험

앱은 localStorage에서 이전 프로젝트를 복원한다.

위험:

- 이전 구조의 프로젝트 데이터가 남아 있으면 새 네비게이션 정책이나 새 기본 블록 설명이 즉시 반영되지 않을 수 있다.
- 사용자가 이미 만든 블록은 기존 제목/본문을 유지한다.

현재 완화:

- `sanitizeProject()`가 기본 구조를 보정한다.

남은 위험:

- 기존 프로젝트의 낡은 텍스트까지 자동 마이그레이션하지는 않는다.
- 필요하면 명시적 마이그레이션 또는 “새 기본 템플릿 적용” 기능이 필요하다.

### 9. 시각적 검증 부족

검증은 빌드, 생성, 문법 검사, HTTP 응답 중심으로 수행했다.

위험:

- 실제 브라우저 픽셀 기준의 겹침, 버튼 잘림, 스크롤 동작, 팝업 렌더링은 자동화 검증되지 않았다.

현재 확인:

- Vite 빌드 통과.
- 생성 앱 빌드 통과.
- 새 창 미리보기 문서 구조/JSON 파싱 확인.
- 서버 200 응답 확인.

남은 위험:

- Playwright 같은 브라우저 자동 검증을 추가해야 UI 겹침과 새 창 렌더링을 더 확실히 잡을 수 있다.

### 10. 한글 인코딩 표시 문제

PowerShell 출력에서 일부 한글이 깨져 보였다.

위험:

- 실제 파일은 UTF-8이어도 터미널 출력이 mojibake로 보이면 사람이 잘못 판단할 수 있다.
- README 일부는 현재 터미널에서 깨져 보였고, 실제 파일 내용도 추가 확인이 필요하다.

현재 처리:

- 이번 요청 범위에서는 README를 직접 수정하지 않았다.
- 새로 작성한 로그와 카탈로그는 UTF-8 텍스트로 작성했다.

남은 위험:

- 모든 Markdown/JS 파일의 인코딩 일관성 검사가 별도로 필요하다.

### 11. 생성 앱 포트 충돌

기존 저장된 `start-site.cmd`는 기본적으로 `5173` 포트를 사용했다.

위험:

- 빌더 앱이 이미 `5173`을 쓰고 있으면 저장된 앱 실행 시 포트 충돌이 날 수 있었다.
- 여러 저장본을 동시에 실행하면 같은 포트 때문에 일반 사용자가 원인을 알기 어려운 실행 실패를 겪을 수 있었다.

현재 수정:

- 저장 시 `vite.config.js`가 `10000~49999` 범위에서 랜덤 포트를 만든다.
- 생성 직전에 `127.0.0.1` 기준으로 해당 포트를 열 수 있는지 확인한다.
- 저장된 앱 실행 스크립트는 `npm run dev -- --port {launchPort} --open`을 사용한다.
- `output/project.interface.json`에도 `launchPort`를 기록한다.

남은 위험:

- 저장 시점에는 사용 가능한 포트였더라도, 나중에 실행하는 순간 다른 프로그램이 같은 포트를 먼저 점유하면 충돌할 수 있다.
- 이 경우 실행 시점에 다시 포트를 탐색하는 launcher 로직이 있으면 더 안전하다.

### 12. 파일 시스템 저장 권한

저장은 바탕화면에 폴더를 만든다.

위험:

- 사용자 권한, 백신, OneDrive 동기화, 경로 길이, 특수문자에 따라 저장이 실패할 수 있다.

현재 완화:

- 파일명 안전화와 중복 폴더명 회피 로직이 있다.

남은 위험:

- 저장 실패 시 복구/재시도 UI는 단순 메시지 수준이다.

### 13. 네트워크 의존성

저장된 앱은 첫 실행 시 `node_modules`가 없으면 `npm install`을 실행한다.

위험:

- 네트워크가 없거나 npm registry 접근이 막히면 일반 사용자가 실행에 실패할 수 있다.

현재 상태:

- 실패 시 cmd에서 메시지와 pause를 제공한다.

남은 위험:

- 완전 오프라인 실행 패키징은 아직 없다.

### 14. 테스트 커버리지 부족

현재 자동 테스트 파일은 없다.

검증은 명령 기반으로 수행했다.

남은 위험:

- 회귀 테스트가 없어서 기능 변경 시 네비게이션, 미리보기, 저장 생성물이 다시 깨질 수 있다.

권장 추가 테스트:

- `blockRuntime` 단위 테스트.
- `siteExport` 생성물 스냅샷 테스트.
- Playwright 기반 새 창 미리보기/저장 앱 렌더링 테스트.
- localStorage 구버전 프로젝트 마이그레이션 테스트.
